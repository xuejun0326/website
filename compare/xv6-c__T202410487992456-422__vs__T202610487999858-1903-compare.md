### 7.1 启动 (`boot`)

**对照判定**: 实现路径分化

**A 仓做法**: 该模块实现了从硬件复位到 C 运行环境的最小启动流程。核心抽象包括汇编入口 `_entry` 与 `start` 函数，通过为每个 CPU 分配独立栈（`stack0[NCPU][4096]`）支持多核，并直接跳转到 `main()` 完成后续初始化。相比 xv6-riscv 原版，代码结构高度相似，但通过平台目录（qemu/visionfive）分离 entry 与 start 文件，保留了跨平台扩展能力。然而，启动流程异常精简——未进行页表、中断、时钟等任何硬件初始化，完全依赖 `main` 内部完成，且存在多处隐式依赖与潜在越界风险。

**B 仓做法**: 启动流程模块实现了从 CPU 复位到内核调度器启动的完整引导序列。核心抽象是两级入口：汇编层 `_entry` 负责架构相关的初始栈设置与特权级配置，C 层 `main()` 顺序初始化内存管理、中断控制器、进程表、文件系统等子系统，最终移交至 `scheduler()`。与 xv6-riscv 原版相比，本项目的 RISC-V 版移除了 machine-mode 过渡代码（依赖 SBI 预置 Supervisor 环境），并新增 LoongArch 架构支持，扩展了 ext4 文件系统和 virtio 块设备驱动。关键数据结构包括 per-CPU 栈数组 `stack0`/`boot_stack`、多核同步标志 `started`，以及各子系统的全局表。

**关键差异点**:

- **多核同步机制** (新增)
